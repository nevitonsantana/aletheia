# Hermes first real microtest — execution contract

Date: 2026-04-28
Issue: #105 — Hermes first real microtest contract
Status: blocked pending explicit human approval

## 1. Purpose

Define the smallest safe contract for the first real Hermes microtest after the sandbox-install diagnostics in Issue #103 / PR #104.

This document is not an execution record. It does not authorize running Hermes. It records the decision points required before any real Hermes task runs.

## 2. Prior evidence

The sandbox-install diagnostics established that:

- Hermes Agent v0.11.0 can be installed in an isolated sandbox home.
- `hermes doctor` runs successfully enough to inspect the environment.
- Agentic Stack can install the `hermes` adapter in a disposable project.
- Agentic Stack `doctor` reports the `hermes` adapter green.
- No real Hermes task has run yet.
- There is no Hermes telemetry yet.

Durable evidence lives in:

- `docs/aletheia/closeouts/2026-04-28-hermes-agentic-stack-sandbox-diagnostics.md`

## 3. Required human decisions before execution

A human must explicitly choose all items below before the microtest starts.

### 3.1 Provider/auth route

Choose one:

- Hermes-supported auth without putting secrets in the repo, such as `hermes auth` / OpenAI Codex auth if available in the sandbox.
- OpenRouter with a temporary key only in the sandbox `.env`.
- Nous Portal auth.
- Local OpenAI-compatible endpoint without external credential.
- Another explicitly named provider/auth route.

No provider secret may be committed, copied into AletheIA, or stored in durable docs.

### 3.2 Disposable working directory

Use a fresh sandbox root matching:

```text
/private/tmp/hermes-real-microtest-YYYYMMDD-HHMMSS
```

Do not execute the microtest from:

- the root of the main AletheIA checkout;
- a dirty local checkout;
- a user home directory;
- any directory containing durable user memory or production credentials.

### 3.3 Exact allowed task

Recommended minimum task:

> Run Hermes once in the disposable sandbox to produce `microtest-output.md` containing only: environment detected, confirmation that the sandbox `AGENTS.md` was read if present, and one recommendation for the next step.

The task must not ask Hermes to fix code, edit the real repository, use credentials, call external production services, start daemons, or operate continuously.

### 3.4 `.agent/` visibility

Recommended default: expose only the `.agent/` generated by Agentic Stack inside the disposable sandbox.

Reason: the goal of this microtest is to verify Hermes + Agentic Stack interaction, not to promote Agentic Stack memory/skills into the real AletheIA repo.

Alternative: run without `.agent/` if the human wants a Hermes-only baseline first.

### 3.5 Success and stop criteria

Success requires all of the following:

- Hermes runs exactly one bounded microtest task.
- `microtest-output.md` is created inside the disposable sandbox.
- Hermes output is clearly separated from Codex observations.
- No file outside the sandbox changes.
- No memory or skill is promoted.
- No gateway, cron, daemon, service, or autonomy is activated.
- No provider secret is written outside the sandbox.

Stop immediately if Hermes:

- asks for a provider/auth path not approved in section 3.1;
- attempts or instructs writing outside the sandbox;
- attempts to install or start gateway, cron, daemon, service, or continuous automation;
- attempts to promote memory or skills;
- requires a secret not already approved for the sandbox;
- expands beyond the exact allowed task.

## 4. Required evidence after execution

If the human approves execution later, the resulting closeout must include:

- exact command used to launch Hermes;
- full sandbox path;
- file tree before and after;
- contents of `microtest-output.md`;
- explicit statement of whether `.agent/` was visible;
- explicit statement of provider/auth path used, without secrets;
- verification that the main AletheIA checkout was not modified;
- verification that real user-level Hermes paths were not modified unexpectedly;
- verdict: passed, blocked, or failed with reason.

## 5. Non-goals

This contract does not authorize:

- production use;
- persistent autonomy;
- gateway or messaging activation;
- scheduled tasks;
- ADR changes;
- promoting Agentic Stack memory/skills into AletheIA;
- treating Codex observations as Hermes telemetry;
- treating a single microtest as readiness for a pilot.
